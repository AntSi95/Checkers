# Checkers

Автономный и расширяемый игровой движок для игры в шашки, написанный на платформе **.NET 10 (C# 14)**. Библиотека предоставляет полный инструментарий для управления партией, контроля состояний и валидации ходов, и спроектирована как независимый компонент для последующего использования в сетевых (Web/API) проектах.

---

## ✨ Ключевые особенности

*   **Иерархическая структура игрового действия (`Turn` -> `Move`):** Жизненный цикл действий строго разграничен. Единицей истории является ход игрока (`Turn`), который инкапсулирует в себе список последовательных перемещений (`Move`). Структура `Move` содержит фиксированные параметры (`From`, `To`, `Target`), являясь техническим воплощением каждого шага. Такая модель позволяет надежно изолировать и проигрывать многоходовые серии прыжков.
*   **Управление статусом и фиксация финала партии:** Движок выступает в роли независимого арбитра. Он определяет текущее состояние игрового процесса (`GameStatus`) и предоставляет актуальный статус. При возникновении бесспорных патовых ситуаций (например, взаимное запирание обеих сторон) движок берет управление на себя и принудительно останавливает партию, фиксируя статус `MutualBlock`.
*   **Обработка сложных механик через конвейер состояний:** Реализация специфических шашечных правил (включая запрет на повторное взятие одной фигуры за ход — «Турецкий удар») построена на системе временных флагов фигуры (`IsCaptured`, `IsPromoted`). Фигуры не удаляются с поля и не превращаются в дамку мгновенно в процессе шага, изменения накапливаются и физически применяются движком в фазе финализации хода (`OnFinalize`) или при непосрдственном запросе в фазе хода (`ProcessStep`).
*   **Единый интерфейс взаимодействия (Фасад):** Все операции по управлению сессией, валидации входящих перемещений, откату шагов (`Undo`) и получению слепков данных для сохранения (`GetSnapshot`) сведены к единому классу-менеджеру — `GameSession`. Это позволяет использовать движок как автономный изолированный компонент в любых сетевых проектах.
*   **Расширяемость через наследование правил:** Движок спроектирован с расчетом на создание кастомных модификаций. Путем наследования и реализации контракта `IRulesStrategy` разработчик может переопределить геометрию доски, условия продвижения фигур и логику завершения хода.
*   **Готовые конфигурации «из коробки»:** Движок содержит предустановленные и полностью протестированные реализации классических шашечных систем (Русские, Английские, Международные). Это позволяет развернуть и запустить полноценную игровую сессию сразу после подключения библиотеки, без необходимости писать собственные стратегии правил.

## 🚀 Быстрый старт

Движок полностью берет на себя создание поля, расстановку фигур и ведение игрового цикла.

### Инициализация игровой сессии
```csharp
using checkers.Core;
using checkers.Core.Rules;
using checkers.GameObjects.Scanning;

// 1. Выбираем стратегию правил и сканер
var rules = new RussianRules();
var scanner = new MoveScanner();

// 2. Запускаем сессию управления игрой
var session = new GameSession(rules, scanner);
```

### Обработка игрового действия
```csharp
var currentSide = session.ActiveSide;

// Получаем список валидных ходов для текущей ситуации (из оптимизированного кэша сессии)
List<Move> validMoves = session.GetValidMoves(currentSide);

if (validMoves.Contains(chosenMove))
{
    // MakeMove сам проверит, завершен ли ход или игроку нужно продолжить серию прыжков
    session.MakeMove(currentSide, chosenMove);
}
```

### Сохранение и восстановление партии
```csharp
// Получение сериализуемого снимка истории ходов
GameSnapshot snapshot = session.GetSnapshot();

// Полное восстановление игрового процесса из истории ходов (автоматически определит активного игрока)
GameSession restoredSession = GameSession.Restore(snapshot.History, rules, scanner);
```

## 📝 Развитие проекта (Технический долг / TODO)
В следующих версиях планируется:
*   **Инверсия управления доской (Фабрика правил):** Полностью отвязать `GameSession` от создания физического поля. Перенести ответственность за порождение инстанса `Chessboard` из конструктора сессии напрямую в фабричные методы `IRulesStrategy`. Это позволит изолировать доску как самостоятельный элемент и передавать в движок кастомные сетки (например, цилиндрические или гексагональные доски) без изменения логики сессии.
*   **Глубокий анализ истории и Приоритет большинства:** В текущей версии движка отсутствует алгоритм построения полного дерева ходов (глубинный анализ). По этой причине специфическое правило международных шашек на **Приоритет большинства** (обязанность выбирать траекторию с максимальным количеством срубленных фигур) временно не реализовано.
*   **Рефакторинг доступа к доске:** перевод `Chessboard` в `internal` и выделение read-only интерфейса `IBoardInspector` для судейских методов правил.
*   **Унификация мутаций:** перенос методов `TurnExecutor` напрямую в контекст управления доской.
